📁 File Server & Shares (GUI)

Shares, NTFS permissions, ABE, quotas, shadow copies — click-by-click.

📌 Is page ka context
KYA
File Server role, shares, NTFS permissions, ABE, quota aur shadow copies setup karna.
KYU
Central storage = backup, access control aur auditing. Local PC storage = data loss risk.
KAISE
Role install → share banao → NTFS perms (least privilege) → ABE + shadow copies. Neeche steps.
⚠️ NTFS vs Share permission conflict mein most restrictive apply hota hai. Plan karke set karo.

Install File Server Role

  1. Server Manager → Manage → Add Roles and Features
  2. Server Roles → expand File and Storage Services → File and iSCSI Services
  3. ✅ File Server, ✅ File Server Resource Manager (FSRM for quota), ✅ DFS Namespaces (optional)
  4. Next → Install

Create Share

  1. File Explorer → folder banao e.g. E:\Data\Finance
  2. Right-click folder → Properties → Sharing tab → Advanced Sharing
  3. ✅ Share this folder → Share name: Finance
  4. Permissions → remove Everyone → Add Authenticated Users = Change → OK
💡 Share level broad rakho (Change), real control NTFS level par karo.

NTFS Permissions (real control)

  1. Folder Properties → Security tab → Advanced
  2. Disable inheritance → "Convert inherited to explicit"
  3. Remove extra groups → Add SG-Finance = Modify, Domain Admins = Full
  4. OK → OK
GroupNTFS
SG-FinanceModify
Domain AdminsFull Control
EveryoneREMOVE

Access Based Enumeration (ABE)

Users ko sirf wohi folders dikhein jinka access hai:

  1. Server Manager → File and Storage Services → Shares
  2. Right-click share → Properties → Settings → ✅ Enable access-based enumeration

Quota & File Screening (FSRM)

  1. Tools → File Server Resource Manager
  2. Quota Management → right-click → Create Quota → path + limit (e.g. 5GB)
  3. File Screening → Create File Screen → block .mp3, .exe etc.

Shadow Copies (Previous Versions)

  1. File Explorer → right-click drive E: → Configure Shadow Copies
  2. Select volume → Enable → Settings → schedule set
💡 Users khud "Previous Versions" tab se restore kar sakte → helpdesk load kam.

Validate

  • Client se: \\FS01\Finance open ho raha?
  • Security tab → Advanced → Effective Access tab se user ki real permission check karo
  • Non-member user ko access deny hona chahiye

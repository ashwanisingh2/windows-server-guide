Production-ready Implementation & Troubleshooting · Greenfield + Brownfield · Ready-to-run PowerShell

Phase 1: Foundation (Day 1-2)

1. Infrastructure Planning

Engineer ko kya karna hai

Deploy se pehle paper/Excel par design lock karo: IP schema, VLAN, domain name, DC IPs, naming.

ItemDecisionExample
IP SchemaPrivate + subnet10.10.0.0/16
Server VLANDedicatedVLAN 10 → 10.10.10.0/24
DomainInternalcorp.company.local
DC IPsStaticDC01=10.10.10.10

Kyu / Benefit

Galat IP/domain design = poora AD rebuild. Ek baar sahi → 5 saal stable.

⚠️ Single-label domain (sirf "company") mat rakho. corp.company.local use karo.

2. Server Install & Hardening

Rename-Computer -NewName "DC01" -Restart
Install-Module PSWindowsUpdate -Force
Get-WindowsUpdate -Install -AcceptAll -AutoReboot
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled True
Rename-LocalUser -Name "Administrator" -NewName "corpadmin"

Kyu / Benefit

Default install = open services, weak defaults. Secure baseline = audit ready.

💡 Core vs Full: DC/DNS/DHCP ke liye Server Core prefer karo (kam attack surface).

3. Static IP & Network

Get-NetAdapter
New-NetIPAddress -InterfaceIndex 5 -IPAddress 10.10.10.10 -PrefixLength 24 -DefaultGateway 10.10.10.1
Set-DnsClientServerAddress -InterfaceIndex 5 -ServerAddresses 10.10.10.10
⚠️ PRODUCTION RISK: Remote IP change + RDP connected + galat gateway = lock out. Console/iLO access ready rakho.

4. Active Directory (First DC)

Install-WindowsFeature AD-Domain-Services -IncludeManagementTools
Install-ADDSForest -DomainName "corp.company.local" -DomainNetbiosName "CORP" `
  -DatabasePath "E:\NTDS" -SysvolPath "E:\SYSVOL" -LogPath "E:\Logs" -InstallDns:$true -Force

Validation

dcdiag /v
Get-ADDomainController
nltest /dsgetdc:corp.company.local
Get-Service NTDS,DNS,Netlogon
Agar galat ho jaye: dcpromo.log check karo (C:\Windows\debug). Demote: Uninstall-ADDSDomainController.

5. Additional DC (DC02)

Install-ADDSDomainController -DomainName "corp.company.local" -InstallDns:$true `
  -Credential (Get-Credential) -Force
repadmin /replsummary
repadmin /showrepl
dcdiag /test:replications
⚠️ DC02 preferred DNS = DC01, secondary = khud. Islanding avoid karo.

Phase 2: Identity & Access (Day 3-4)

6. OU Design & Delegation

New-ADOrganizationalUnit -Name "CORP-HQ" -Path "DC=corp,DC=company,DC=local"
New-ADOrganizationalUnit -Name "Users" -Path "OU=CORP-HQ,DC=corp,DC=company,DC=local"
redirusr "OU=Users,OU=CORP-HQ,DC=corp,DC=company,DC=local"
redircmp "OU=Computers,OU=CORP-HQ,DC=corp,DC=company,DC=local"

Delegation: dsa.msc → OU → Delegate Control → helpdesk group → reset passwords.

7. User & Group Strategy (AGDLP)

New-ADUser -Name "Amit Sharma" -SamAccountName "amit.sharma" `
  -UserPrincipalName "amit.sharma@corp.company.local" `
  -Path "OU=Users,OU=CORP-HQ,DC=corp,DC=company,DC=local" `
  -AccountPassword (Read-Host -AsSecureString) -ChangePasswordAtLogon $true -Enabled $true
New-ADGroup -Name "SG-Finance" -GroupScope Global -GroupCategory Security
Add-ADGroupMember -Identity "SG-Finance" -Members "amit.sharma"
💡 AGDLP: Accounts → Global → Domain Local → Permission. Resource perms Domain Local group ko do.

8. Group Policy Baseline

New-GPO -Name "Baseline-Security" | New-GPLink -Target "OU=CORP-HQ,DC=corp,DC=company,DC=local"

⚠️ USB block / AppLocker domain-wide directly mat lagao. Pehle pilot OU (5-10 machines), phir rollout.

9. Home Folders & Redirection

New-Item "E:\Users" -ItemType Directory
New-SmbShare -Name "Users$" -Path "E:\Users" -ChangeAccess "CORP\Domain Users"

GPO → User → Folder Redirection → Documents → \\FS01\Users$\%username%

Phase 3: Network Services (Day 5-6)

10. DNS Enterprise Config

Add-DnsServerPrimaryZone -NetworkID "10.10.10.0/24" -ReplicationScope "Forest"
Add-DnsServerConditionalForwarderZone -Name "partner.com" -MasterServers 8.8.8.8
Set-DnsServerScavenging -ScavengingState $true
Add-DnsServerForwarder -IPAddress 8.8.8.8,1.1.1.1
Resolve-DnsName dc01.corp.company.local
dcdiag /test:dns
⚠️ Scavenging galat config = valid records delete. Intervals samajh ke set karo.

11. DHCP Scope Design

Install-WindowsFeature DHCP -IncludeManagementTools
Add-DhcpServerInDC -DnsName dc01.corp.company.local -IPAddress 10.10.10.10
Add-DhcpServerv4Scope -Name "Clients-VLAN20" -StartRange 10.10.20.50 -EndRange 10.10.20.250 -SubnetMask 255.255.255.0
Set-DhcpServerv4OptionValue -ScopeId 10.10.20.0 -Router 10.10.20.1 -DnsServer 10.10.10.10,10.10.10.11
Add-DhcpServerv4Failover -Name "DHCP-FO" -PartnerServer dc02.corp.company.local -ScopeId 10.10.20.0 -LoadBalancePercent 50
Get-DhcpServerv4ScopeStatistics
Get-DhcpServerInDC
⚠️ Rogue DHCP = galat IP/gateway. Switch par DHCP snooping enable karo.

wbadmin start systemstatebackup -backuptarget:F: -quiet

Brownfield Troubleshooting

Scenario 1: User login nahi kar pa raha

Get-ADUser amit.sharma -Properties LockedOut,PasswordExpired,Enabled
Search-ADAccount -LockedOut
w32tm /stripchart /computer:dc01 /samples:3
klist
nltest /dsgetdc:corp.company.local
gpresult /r

Logs: Event 4625 (failed logon), 4740 (lockout). Fix: Unlock-ADAccount; time skew >5min = Kerberos fail → w32tm /resync.

Escalation: Multiple users locked simultaneously = possible attack → senior.

Scenario 2: GPO apply nahi ho rahi

gpupdate /force
gpresult /r
gpresult /h gpo.html
repadmin /showrepl

Cause: Security filtering, WMI filter, block inheritance, not domain-joined, SYSVOL replication pending.

Escalation: SYSVOL replication broken → senior (DFSR recovery).

Scenario 3: DHCP se IP nahi mil rahi

Get-DhcpServerInDC
Get-DhcpServerv4ScopeStatistics
Get-Service DHCPServer
ipconfig /release; ipconfig /renew

Cause: Scope full, service stopped, unauthorized, rogue DHCP, missing relay (ip-helper) on VLAN.

Escalation: Rogue DHCP detected → network + security team.

Scenario 4: File share access denied

whoami /groups
icacls "E:\Data\Finance"
net use \\FS01\Data

Cause: NTFS vs Share conflict (most restrictive wins), explicit Deny, group change (re-login needed), token bloat, ABE. Use Effective Access tab.

Escalation: Token size issue (1000+ groups) → AD cleanup project.

Scenario 5: AD replication fail

repadmin /showrepl
repadmin /replsummary
dcdiag /test:replications
dcdiag /test:dns
portqry -n dc02 -p both -e 135

Cause: DNS fail (most common), time skew, firewall (RPC/135), KCC, USN rollback, lingering objects.

⚠️ USN rollback (DC wrongly restored from snapshot) = replication permanently breaks → rebuild DC. Escalation immediately.

Scenario 6: DC crash restore

Recoverable: DSRM boot → system state restore. Objects deleted: AD Recycle Bin restore. Unrecoverable: metadata cleanup + seize FSMO.

Get-ADObject -Filter 'isDeleted -eq $true' -IncludeDeletedObjects | Restore-ADObject
Move-ADDirectoryServerOperationMasterRole -Identity DC02 -OperationMasterRole 0,1,2,3,4 -Force
⚠️ FSMO seize sirf jab original DC kabhi wapas nahi aayega. Seize ke baad purana DC network par mat lao. Senior approval mandatory.

Scenario 7: Fast onboarding

Import-Csv "C:\new_users.csv" | ForEach-Object {
  New-ADUser -Name "$($_.First) $($_.Last)" -SamAccountName "$($_.First).$($_.Last)" `
    -UserPrincipalName "$($_.First).$($_.Last)@corp.company.local" `
    -Path "OU=Users,OU=CORP-HQ,DC=corp,DC=company,DC=local" `
    -AccountPassword (ConvertTo-SecureString $_.Password -AsPlainText -Force) `
    -ChangePasswordAtLogon $true -Enabled $true
  Add-ADGroupMember -Identity "SG-$($_.Dept)" -Members "$($_.First).$($_.Last)"
}

CSV: First,Last,Dept,Password. Test 1 user first; add -WhatIf for dry-run.

Scenario 8: Security audit fail

Search-ADAccount -AccountInactive -TimeSpan 90.00:00:00 | Disable-ADAccount
Get-ADUser -Filter {PasswordNeverExpires -eq $true} -Properties PasswordNeverExpires
Get-ADGroupMember "Domain Admins"

Fix: Password policy enforce, inactive disable, privileged access review, audit policy GPO, patch compliance.

Escalation: Active compromise (unexpected Domain Admin) → security/IR team immediately.

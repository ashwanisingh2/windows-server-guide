🖥️ GUI Setup Guide

Active Directory + DHCP setup using Server Manager — bina PowerShell, sirf clicks. Step-by-step Hinglish.

📌 Is page ka context
KYA
Server ko Domain Controller banana + DHCP role setup karna (GUI se).
KYU
AD = centralized identity, DHCP = auto IP. Inke bina network aur user management impossible.
KAISE
Server Manager → Add Roles → Promote to DC → DHCP install → authorize → scope. Neeche step-by-step.
⚠️ PRODUCTION: Pehle snapshot/backup lo · Console/iLO access ready rakho (RDP gir sakta hai) · Off-hours/maintenance window mein karo.

Open Server Manager

Start menu → Server Manager   ya   Win + R → type servermanager → Enter.

Ye central dashboard hai jahan se saare roles install/manage hote hain.

AD Step 1: Install Role

  1. Server Manager → top-right Manage → Add Roles and Features
  2. "Before You Begin" → Next
  3. Installation Type → Role-based or feature-based installation → Next
  4. Server Selection → apna server select → Next
  5. Server Roles → ✅ tick Active Directory Domain Services
  6. Popup "Add features?" → Add Features
  7. Next → Next → Next → Install
  8. Install hone do (abhi reboot nahi hoga)

AD Step 2: Promote to Domain Controller

  1. Install ke baad Server Manager mein top par yellow ⚠️ flag dikhega → click
  2. Promote this server to a domain controller
  3. Deployment Configuration → Add a new forest (nayi company / pehla DC)
  4. Root domain name: corp.company.local → Next
  5. Domain Controller Options → functional level default → DSRM password daalo (yaad rakho!) → Next
  6. DNS Options → warning ignore → Next
  7. Additional Options → NetBIOS auto (CORP) → Next
  8. Paths → NTDS/SYSVOL location (default ya alag disk) → Next
  9. Review → Next
  10. Prerequisites Check → green tick → Install
  11. Server automatic reboot hoga
💡 TIP: Production mein NTDS/SYSVOL alag disk (E:) par rakho — better performance + recovery.

AD Step 3: Validate

Reboot ke baad Server Manager → Tools menu mein ab ye dikhega:

  • Active Directory Users and Computers — users/OU manage
  • Active Directory Sites and Services — replication
  • DNS — zones auto-create ho gaye

Kholo aur confirm domain corp.company.local dikh raha hai. DNS console mein Forward Lookup Zone bani honi chahiye.

DHCP Step 1: Install Role

  1. Server Manager → Manage → Add Roles and Features
  2. Next → Next → Next → Server Roles → ✅ DHCP Server
  3. "Add Features" → Add Features
  4. Next → Next → Install

DHCP Step 2: Authorize in AD

⚠️ KYU: Bina authorization ke Windows DHCP leases nahi deta (rogue DHCP protection).
  1. Install ke baad yellow ⚠️ flag → click
  2. Complete DHCP configuration
  3. Authorization step → domain admin credentials → Commit → Close

DHCP Step 3: Create Scope

  1. Server Manager → Tools → DHCP
  2. Expand: Server name → IPv4 → right-click → New Scope
  3. Name: LAN-Clients → Next
  4. IP Range: Start 192.168.1.100, End 192.168.1.200, Subnet 255.255.255.0 → Next
  5. Exclusions: 192.168.1.1 to 192.168.1.20 (servers/printers ke liye) → Next
  6. Lease duration: 8 days (default) → Next
  7. "Configure options now?" → Yes → Next
  8. Router/Gateway: 192.168.1.1 → Add → Next
  9. DNS: Domain corp.company.local, Server IP 192.168.1.10 → Next
  10. WINS → skip → Next
  11. Activate scope now? → Yes → Next → Finish

DHCP Step 4: Validate

  • DHCP console → Scope → Address Leases mein clients dikhenge
  • Client PC par CMD: ipconfig /release phir ipconfig /renew
  • Client ko IP 192.168.1.100-200 range se milna chahiye, DNS = 192.168.1.10
💡 Reservation: Printer/server ko fixed IP dena ho → Scope → Reservations → right-click → New Reservation → MAC + IP daalo.

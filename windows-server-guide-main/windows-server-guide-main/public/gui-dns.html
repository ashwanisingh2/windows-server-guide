🌐 DNS Configuration (GUI)

Forward/Reverse zones, records, forwarders — DNS Manager se, click-by-click.

📌 Is page ka context
KYA
DNS zones (forward/reverse), records (A/CNAME/MX), forwarders aur scavenging configure karna.
KYU
DNS AD ki backbone hai. Iske bina domain, internet aur email kuch nahi chalta.
KAISE
DNS Manager (dnsmgmt.msc) se zones banao, records add karo, forwarders set karo. Neeche steps.
⚠️ DNS galat config = domain, internet, email sab fail. Change se pehle current records note karo.

Open DNS Manager

Server Manager → Tools → DNS   ya   Win+R → dnsmgmt.msc

AD install ke baad Forward Lookup Zone (corp.company.local) already bana hota hai.

Forward Lookup Zone banao

  1. DNS Manager → expand server → right-click Forward Lookup Zones → New Zone
  2. Welcome → Next
  3. Zone Type → Primary zone + ✅ Store zone in Active Directory → Next
  4. Replication scope → To all DNS servers in this domain → Next
  5. Zone name → corp.company.local → Next
  6. Dynamic Update → Allow only secure dynamic updates → Next → Finish
💡 AD-integrated + secure updates = best practice (auto replicate, secure).

Reverse Lookup Zone banao (PTR ke liye)

  1. Right-click Reverse Lookup Zones → New Zone
  2. Primary + Store in AD → Next
  3. IPv4 Reverse Lookup Zone → Next
  4. Network ID: 192.168.1 → Next
  5. Secure dynamic updates → Next → Finish
⚠️ KYU: Reverse zone na ho to email delivery (PTR check) aur security tools fail karte hain.

Records add karo (A / CNAME / MX)

Forward zone par right-click → record type choose:

RecordKabExample
AHostname → IPfs01 → 192.168.1.20
CNAMEAliasintranet → fs01.corp.company.local
MXMail servermail.corp.company.local (priority 10)
  1. Zone right-click → New Host (A)
  2. Name + IP daalo → ✅ "Create associated PTR record" → Add Host

Forwarders (internet resolution)

  1. DNS Manager → right-click server name → Properties
  2. Forwarders tab → Edit
  3. Public DNS daalo: 8.8.8.8, 1.1.1.1 → OK

Conditional Forwarder (external domain)

  1. Right-click Conditional Forwarders → New Conditional Forwarder
  2. Domain: partner.com, IP: partner's DNS → OK

Scavenging (stale records cleanup)

  1. Right-click server → Properties → Advanced tab
  2. ✅ Enable automatic scavenging → interval 7 days → OK
⚠️ Scavenging aggressive ho to valid records delete ho jaate. Intervals samajh ke set karo.

Validate

  • CMD: nslookup corp.company.local
  • nslookup fs01.corp.company.local
  • nslookup google.com (forwarder test)

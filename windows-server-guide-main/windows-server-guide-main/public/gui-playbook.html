📕 IT Infrastructure Playbook

System Administrator ka complete playbook — naye organization mein deploy karne ke liye. 12 phases, troubleshooting framework, deliverables. Hinglish.

📌 Is page ka context
KYA
End-to-end IT infra deployment: network, servers, AD, email, clients, security, backup, docs, vendor.
KYU
Bina structured plan ke deployment chaos ban jaata hai. Phase-wise approach = predictable, low-risk.
KAISE
10-week phased rollout. Har phase: kyu zaroori, benefit, priority, real example. Neeche detail.

Priority Legend

P0 Critical = bina iske kuch nahi chalega   P1 Important = jaldi chahiye   P2 Standard = plan karke karo

💡 Decision tree (on-prem vs cloud vs hybrid): Data sovereignty/legacy app? → On-prem. Scalability/remote-first? → Cloud. Dono mix/migration phase? → Hybrid.

Phase 1: Discovery & Planning P0 (Week 1)

KYU
Bina existing setup samjhe naya banaoge to conflict + rework. "Pehle dekho, phir decide karo."
BENEFIT
Accurate budget, realistic timeline, no surprises baad mein.
  • Infra audit: Lansweeper / Advanced IP Scanner se network discover karo — kya devices, OS, IPs hain
  • Topology map: switches, routers, servers, links document karo
  • Stakeholder interviews: HR, Finance, Dev team, Management — unki needs samjho
  • Risk assessment: single points of failure, EOL hardware, security gaps

Real example: Nayi company join ki — pehle 1 week sirf audit. Pata chala 2 servers warranty-expired the, ek switch single point of failure tha. Plan usi hisaab se bana.

Phase 2: Core Infrastructure P0 (Week 2-3)

KYU
Network + AD = foundation. Galat foundation par sab kuch unstable rahega.
BENEFIT
Stable, secure, scalable base — 5 saal tak problem nahi.

  • Remote access: VPN (IKEv2/SSTP enterprise; WireGuard fast/modern)
  • Firewall: VLAN-to-VLAN rules, least privilege segmentation

2.2 Windows Server Core

Roles: AD DS, DNS, DHCP, File Server, Print Server. Hardening baseline. Updates: WSUS (on-prem control) ya WUfB (cloud-managed).

2.3 Active Directory & Identity

Single forest/domain (most orgs). OU = Geographic + Functional. GPO baseline (security, desktop, updates). Naming convention (first.last). Service accounts documented + least privilege.

Phase 3: Network Services P0 (Week 4)

KYU
DNS/DHCP/remote access ke bina users connect aur kaam nahi kar sakte.
BENEFIT
Smooth connectivity, auto IP, secure remote work.
  • DNS: internal zones, conditional forwarders, split-brain (internal/external), DNSSEC consider
  • DHCP: scope per VLAN, reservations (servers/printers), failover (hot-standby/load balance), options (DNS/domain/NTP)
  • Remote access: VPN gateway + NPS/RADIUS + MFA (Azure AD / Duo)

Phase 4: Email & Collaboration P1 (Week 5)

KYU
Email = business communication backbone. Down = company ruk jaati hai.
BENEFIT
Reliable email, collaboration, mobility.
OptionKab choose karein
Exchange On-PremFull control, data sovereignty, compliance strict. DAG for HA.
Microsoft 365Most orgs — no server maintenance, Teams/SharePoint included. E3/E5/F3 licenses.
HybridMigration phase ya mix — Azure AD Connect se sync.
Google WorkspaceStartups/Google-first orgs. Domain verify, dual-delivery routing.
💡 Decision: Naya org, <500 users? → M365 (sabse simple). Strict on-prem compliance? → Exchange. Existing on-prem + cloud move? → Hybrid.

Phase 5: Client Management P1 (Week 6)

KYU
100s of endpoints manually manage = impossible. Standardize + automate.
BENEFIT
Consistent, secure, fast deployment, easy support.
  • Windows 10/11: deploy via Intune (modern/cloud) ya SCCM/MDT/WDS (on-prem). BitLocker, app deploy, WUfB
  • macOS: Apple Business Manager + MDM (Jamf ya Intune). FileVault. AD join alternative: Jamf Connect / NoMAD
  • macOS Server (rare): Profile Manager, file sharing (SMB), caching server — mostly Jamf ne replace kar diya

Phase 6: Developer Network P1 (Week 7)

KYU
Dev team ko isolated, flexible environment chahiye — production ko risk nahi.
BENEFIT
Safe experimentation, fast CI/CD, production protected.
  • Dev VLAN: isolated, proxy/content filter, internal registries (NuGet/NPM/Docker)
  • Code repo: GitLab / Azure DevOps / GitHub Enterprise — branch policies, code review, backup
  • Test env: VM infra, sandbox network (prod se isolated), anonymized DB copies

Phase 7: Monitoring & Performance P1 (Week 8)

KYU
Problem pata chalne se pehle fix = proactive. Reactive = downtime + blame.
BENEFIT
Early warning, trend analysis, capacity planning.
  • System: baselines, SIEM (Sentinel/Splunk/Elastic), SNMP (PRTG/Zabbix/Nagios), AD replication, DHCP utilization
  • Network: bandwidth, latency, Wireshark (packet capture), NetFlow
  • Capacity: disk/memory/CPU trends, growth projection, license utilization

Phase 8: Security & Compliance P0 (Ongoing)

KYU
Security afterthought nahi hai. Ek breach = data loss + reputation + legal.
BENEFIT
Protected data, compliance pass, business trust.
  • Identity: strong password policy, MFA everywhere, PIM/JIT privileged access, regular access reviews
  • Endpoint: Defender for Endpoint, macOS baseline, EDR, vulnerability scan (Nessus/Qualys)
  • Network: segmentation verify, firewall audit, IDS/IPS, DLP
  • Compliance: audit logging, data retention, GDPR/SOC2/ISO as needed

Phase 9: Backup & Disaster Recovery P0 (Week 9)

KYU
Ransomware/hardware fail/human error — kuch bhi ho sakta. Backup = last line of defense.
BENEFIT
Business continuity, data recovery, compliance.
  • 3-2-1 rule: 3 copies, 2 media, 1 offsite
  • Backup: System State (AD), full server (Veeam/Azure Backup/Commvault), Exchange mailbox, file server, SQL/DB
  • DR: DR site (Azure Site Recovery/warm standby), RTO/RPO define, quarterly failover testing, runbooks
  • Business continuity: critical services priority, communication plan, alternative workspace
⚠️ Untested backup = no backup. Quarterly restore drill mandatory.

Phase 10: Documentation P1 (Week 10)

KYU
"Jo document nahi hai wo exist nahi karta." Engineer chala gaya to knowledge bhi gaya.
BENEFIT
Knowledge transfer, faster troubleshooting, audit proof.
  • Network: physical + logical topology, IP schema, VLAN table, port mapping
  • System: server inventory (HW/OS/roles/warranty), app inventory, service accounts, password vault (KeePass/Bitwarden/Key Vault)
  • Procedural: runbooks, troubleshooting guides, change management, onboarding/offboarding
  • Asset: HW lifecycle, license compliance, procurement workflow, warranty tracking
💡 Documentation kaise banayein: Documentation Methodology page dekho.

Phase 11: User Support & Training P2 (Ongoing)

KYU
Trained users = kam tickets. Untrained users = phishing victims + repeated issues.
BENEFIT
Lower support load, fewer security incidents, happy users.
  • Help desk: ticketing (ServiceNow/Jira/Freshservice), SLA, self-service KB, escalation matrix
  • Training: security awareness (phishing/passwords), software usage, new-employee IT orientation, refreshers
  • Policies: Acceptable Use, clean desk, data handling, incident reporting

Phase 12: Vendor Management P2 (Ongoing)

KYU
Hardware/software/ISP vendors manage karna bhi IT ka kaam. Support contract na ho to outage lamba.
BENEFIT
Fast support, planned refresh, no surprise EOL.
  • Coordination: HW support (Dell ProSupport/HP Care Pack), SW (MS Unified Support), ISP, cloud vendors
  • Lifecycle: warranty tracking, refresh cycle (desktop 4yr, server 5yr, network 7yr), EOL/EOS, disposal (data destruction certificate)

Troubleshooting Framework (If This, Then That)

TierKya karo
1. InformationSymptoms poochho, recent changes?, kitne affected?, error screenshots
2. IsolationScope (single/dept/company), OSI layer identify, last working state
3. ResolutionStandard fixes (restart, gpupdate, ipconfig renew), logs check, escalation criteria
4. DocumentationRoot cause + fix steps + prevention + KB update
IF single user affected      -> client-side (profile, network, app)
IF whole department          -> switch/VLAN/GPO/share
IF company-wide              -> core service (DNS/AD/DHCP/internet/firewall)
IF "worked yesterday"        -> recent change/update/patch dekho

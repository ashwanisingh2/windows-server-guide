📦 Core Services Reference

Windows Server ke must-have roles — har ek ka purpose, bina iske kya hoga, real use case, aur priority tier.

💡 Note: "Alternative koi nahi" technically galat hai — har cheez ka koi alternative hota hai. Yahan likha hai Windows ecosystem mein iski jagah lena kyu mushkil hai.

Priority Ranking (ek nazar mein)

TierMatlabServices
Tier 1Must-have, bina iske company chal hi nahi saktiAD DS, DNS, DHCP, File Server/SMB, Group Policy, Defender, Firewall, Backup, VSS
Tier 2Important, zyaadatar orgs meinAD CS, NPS, RRAS/VPN, DFS, IIS, Hyper-V, Task Scheduler, PowerShell/DSC, Event Viewer/PerfMon, RDS, WinRM, BitLocker, AppLocker
Tier 3Nice-to-have / specific needDirectAccess, Storage Spaces/Replica, iSCSI Target, Containers, Admin Center, Application Server

Tier 1 — Must Have (bina iske company down)

ServiceKya hai / PurposeBina iske kya hogaReal use case
AD DSCentral identity directory — users, computers, auth, SSOHar PC alag login, no SSO/policy, 500 users manage impossibleEmployee join → ek account, sab apps/shares/PCs login
DNSNaam → IP resolution; AD ki backboneAD hi nahi chalega, internet/email/apps failfileserver type karo → IP resolve → share khule
DHCPAuto IP/gateway/DNS assignment500 devices manual IP = conflicts, nightmareNaya laptop laga → turant IP + settings auto
File Server / SMBCentral file storage + sharingData PCs par bikhra, no backup/access control\\FS01\Finance → team shared docs, permission-controlled
Group PolicyCentralized config/security enforcement100 PCs manually = impossible, inconsistentEk policy → sab PCs par password rule + USB block
Windows DefenderBuilt-in antivirus / EDRMalware/ransomware open doorReal-time scan, ransomware se files protect
Windows FirewallHost-based traffic controlServer har port par exposed, lateral attack easySirf zaroori ports allow (RDP, SMB), baaki block
Server Backup / WBAdminSystem state + full server backupCrash/ransomware = permanent data lossDC crash → system state restore → AD wapas
VSS (Shadow Copy)Point-in-time snapshots, open-file backupOpen files backup nahi, no "previous versions"File delete → "Previous Versions" se restore
⚠️ Kyu foundation: AD ke bina GPO, file perms (Kerberos), AD CS, NPS chal hi nahi sakte. DNS na ho to AD locate hi nahi hoga.

Dependency Chain (kya kis par depend karta hai)

DNS  <--- AD DS <--- Group Policy, File perms (Kerberos), AD CS, NPS
 |
 +-- DHCP (clients ko DNS deta hai)

AD DS ---> foundation for: Certificates (AD CS), RADIUS (NPS),
           VPN auth (RRAS), DFS, RDS licensing
Bottom line: DNS + AD DS sabse critical — ye girein to poora domain down. Inke upar baaki sab build hota hai.
